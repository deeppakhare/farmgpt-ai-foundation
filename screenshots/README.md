## ➕ How to add a new screenshot

### 1. Capture

- **macOS:** `Cmd + Shift + 4` (area) or `Cmd + Shift + 5` (window)
- **Windows:** `Win + Shift + S` (Snipping Tool)
- **Linux:** `gnome-screenshot -a` or Flameshot
- **Full-page browser capture:** Chrome DevTools → `Cmd/Ctrl + Shift + P` → *"Capture full size screenshot"*

### 2. Optimize (optional but recommended)

Large PNGs bloat the repo. Compress before committing:

- [tinypng.com](https://tinypng.com) — drag & drop, free
- [squoosh.app](https://squoosh.app) — convert to `.webp` for even smaller files
- CLI: `pngquant --quality=65-85 screenshot.png`

Target: **under 500 KB per image**.

### 3. Drop it in this folder

Save the file into `screenshots/` with a clear kebab-case name:

```
screenshots/
├── logo.png
├── dashboard.png
└── disease-scanner.png
```

### 4. Reference it in the README

Use a **relative path** so it works on GitHub, npm, and forks:

```markdown
![Dashboard](./screenshots/dashboard.png)
```

Centered with a caption:

```html
<p align="center">
  <img src="./screenshots/dashboard.png" alt="FarmGPT Dashboard" width="800" />
  <br />
  <em>Command Center — weekly briefing generated by Gemini 3 Flash</em>
</p>
```

Side-by-side comparison:

```html
<p align="center">
  <img src="./screenshots/chat.png" width="45%" alt="Chat" />
  &nbsp;&nbsp;
  <img src="./screenshots/disease-scanner.png" width="45%" alt="Disease scanner" />
</p>
```

Collapsible gallery (keeps the README short):

```markdown
<details>
<summary>📷 More screenshots</summary>

![Weather](./screenshots/weather.png)
![Market](./screenshots/market.png)
![Farm Planner](./screenshots/farm-planner.png)

</details>
```

### 5. Push to GitHub

```bash
git add screenshots/
git commit -m "docs(screenshots): add dashboard and chat screenshots"
git push
```

That's it — the images render automatically in the README on GitHub. ✅

## 💡 Tips for great README screenshots

- Use **dark mode** consistently — matches the FarmGPT visual identity.
- Hide personal data (real emails, tokens, farm names) before capturing.
- Prefer **real data** over Lorem Ipsum — it looks more trustworthy.
- Keep the browser chrome (URL bar) **out** of the shot unless the URL is the point.
- Use a consistent **1600×1000** viewport for desktop shots so the gallery looks uniform.
- Add a subtle drop shadow with [shots.so](https://shots.so) or [screenshot.rocks](https://screenshot.rocks) for a polished feel.
